# Test Spec

## テスト戦略
- ドメインルールはユニットテスト
- ユースケース遷移はアプリケーションテスト
- API は将来統合テストで補完

## 単体テスト観点
- 15分刻み検証
- 4段階承認必須
- 承認権限チェック
- 最終承認で残数消費

## 結合テスト観点
- 申請作成 -> 承認4回 -> Approved
- Reject で遷移停止
- Cancel で待機ステップスキップ

## 境界値テスト
- 15分ちょうど / 14分 / 16分
- ルート段数 4, 20, 21
- 日単位 465分 / 930分

## 異常系テスト
- ルート未設定申請
- 不正ユーザー承認
- 存在しない requestId

## 負荷テスト方針
- 一覧取得 API の1分間スループットを測定
- 目標: 95パーセンタイルで5秒以内
