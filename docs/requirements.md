# 承認決裁システム（休暇申請機能）要件定義書

## 1. 概要

本システム(ApproveFlow)は、休暇申請および承認・決裁フローを電子化することを目的とする。  
PoCでは休暇申請機能に限定し、将来的に他申請（指導受け等）へ拡張可能な構造とする。

---

## 2. システム構成

| 項目 | 内容 |
|------|------|
| フロントエンド | Next.js |
| バックエンド | ASP.NET Core Web API (.NET 7) |
| データベース | インメモリ + RDB（MySQL想定）※切り替え可能 |
| 通知 | SMTP または Microsoft Graph API ※切り替え可能 |

---

## 3. ユーザー種別

| 種別 | 説明 |
|------|------|
| 申請者 | 休暇申請を行うユーザー |
| 承認者 | 承認フローに基づき承認を行うユーザー |
| 管理者 | 全データ閲覧・編集・設定可能 |

---

## 4. 機能要件

### 4.1 休暇申請機能

#### ■ 入力項目
- 申請者
- 休暇種別
- 取得単位
  - 時間単位（15分単位）
  - 日単位（1日）
- 取得日時（開始・終了）
- 理由
- 添付ファイル（任意）

#### ■ 業務ルール
- 1日 = 7時間45分（465分）
- 時間単位は15分刻み
- 日単位は1日単位のみ入力可能

---

### 4.2 休暇残数管理

#### ■ 加算ルール
- 毎月2日付与

#### ■ 繰越ルール
- 基準日（3/31）時点で最大30日まで繰越（基準日および日数設定値は個人ごとに変更可能）
- 超過分は切り捨て

#### ■ 管理内容
- 総付与日数
- 使用日数
- 残日数

---

### 4.3 承認ワークフロー

#### ■ 承認構成
- 最小：4名
- 最大：6名以上（拡張可能）
- 申請者ごとに個別設定可能

#### ■ 承認ステップ
1. 申請
2. 承認（複数段階）
3. 決裁（最終承認）

#### ■ ステータス
- Draft（下書き）
- Submitted（申請中）
- InReview（承認中）
- Approved（承認済）
- Rejected（差戻し）
- Cancelled（取消）

#### ■ 操作
- 承認
- 差戻し（前段階または指定）
- 取消（申請者のみ）

---

### 4.4 承認ルート設定

#### ■ 設定内容
- 承認者氏名
- 承認順序
- 通知メールアドレス
- 通知メール文面（編集可能）

#### ■ 設定単位
- 申請者ごと

---

### 4.5 通知機能

#### ■ 通知タイミング
- 申請時
- 承認依頼時
- 承認完了時
- 差戻し時

#### ■ 通知内容
- 件名・本文テンプレート編集可能
- 差し込み項目（例）
  - 申請者名
  - 日時
  - 承認リンク

---

### 4.6 閲覧・権限管理

| 機能 | 申請者 | 承認者 | 管理者 |
|------|--------|--------|--------|
| 自分の申請閲覧 | ○ | - | ○ |
| 承認対象閲覧 | - | ○ | ○ |
| 全件閲覧 | - | - | ○ |
| 編集 | 自分のみ | - | ○ |

---

### 4.7 印刷機能

#### ■ 対象
- 承認完了済データ

#### ■ 出力形式
- 一覧表形式
- 定型帳票形式（個票）

#### ■ 技術方式
- HTMLテンプレート → PDF変換

#### ■ 条件
- 年度指定（4月〜翌年3月）と暦年（1月〜12月）を選択できる

---

### 4.8 集計・ダッシュボード

#### ■ 表示内容
- 残休暇日数
- 月別取得状況
- 年度取得状況

#### ■ 表示形式
- グラフ（棒・円）
- 数値サマリ

#### ■ 対象範囲
- 年度指定可能（4月〜翌年3月）と暦年（1月〜12月）を選択できる

---

## 5. 非機能要件

### 5.1 パフォーマンス
- 通常操作：3秒以内
- 一覧取得：5秒以内

### 5.2 セキュリティ
- 認証必須（JWT or セッション）
- ロールベースアクセス制御
- 操作ログ保存

### 5.3 可用性
- 業務時間内（平日9:00〜18:00）稼働

## 8. 開発ステップ（PoC）

1. 休暇申請登録API
2. 承認フロー（固定4段階）
3. 承認・差戻し機能
4. 残日数計算
5. UI（申請・承認画面）
6. 印刷（PDF）
7. ダッシュボード

---

## 9. 成功基準（PoC）

- 申請〜承認〜完了まで一連の流れが動作する
- 残休暇計算が正しく行われる
- 承認ルートが変更可能
- 印刷帳票が出力できる
